TRINITY

Manual de Usuario

Sistema web de gestión para salón de belleza

Versión académica y demostrativa

1. Introducción

TRINITY es una plataforma diseñada para centralizar la operación de un salón de belleza. Permite gestionar clientes, servicios, citas, usuarios y reportes desde una sola interfaz.

2. Inicio de sesión

Para entrar al sistema:

  1. Abra la aplicación en el navegador.
  2. Ingrese su correo electrónico y contraseña.
  3. Presione Iniciar sesión.
Si el usuario está inactivo o las credenciales son incorrectas, el sistema mostrará un mensaje de error.

3. Roles del sistema

Rol Acceso principal
Administrador Control total del sistema, usuarios, servicios y reportes.
Recepción Clientes, citas, servicios y reportes operativos.
Estilista Consulta de citas asignadas y actualización de estado o notas.

4. Módulos disponibles

Dashboard

Resume métricas generales del salón, citas recientes, ingresos y desempeño del equipo.

Clientes

Permite registrar, editar, consultar y eliminar clientes cuando no tienen citas activas que lo impidan.

Servicios

Administra el catálogo de servicios, duración, precio, categoría y estado activo o inactivo.

Citas

Gestiona la agenda, asigna estilistas, vincula múltiples servicios y controla estados de atención.

Usuarios

Administra cuentas del sistema, roles, contraseñas y estado de acceso.

Reportes

Presenta indicadores y tablas exportables para ingresos, clientes frecuentes y comportamiento de citas.

5. Flujo básico de uso

  1. Registrar servicios y usuarios necesarios.
  2. Crear clientes.
  3. Programar citas seleccionando estilista, fecha y servicios.
  4. Actualizar el estado de la cita durante la atención.
  5. Consultar reportes para seguimiento del negocio.

6. Recomendaciones de operación

  • Verifique los datos del cliente antes de guardar.
  • Confirme que la cita tenga al menos un servicio asociado.
  • No desactive ni elimine al último administrador activo.
  • Use los reportes para revisar ingresos y servicios más solicitados.

7. Alcance actual y pendientes

El sistema ya cubre el flujo principal de operación del proyecto académico.

También existe estructura inicial en base de datos para:

  • inventario
  • ventas y POS
  • notificaciones

Estos módulos todavía no cuentan con interfaz operativa completa.

8. Soporte técnico

Si el sistema no carga correctamente, revise que el backend, el frontend y la base de datos local estén ejecutándose, y confirme que las variables de entorno estén configuradas.

Para entornos locales, el archivo README.md incluye los comandos necesarios para levantar el proyecto.
